# Contributing to Apollo

Thanks for your interest in contributing to Apollo!

## How to Contribute

1. **Fork** the repo
2. **Create a branch** for your change (`git checkout -b feat/my-feature`)
3. **Make your changes** — keep them focused and minimal
4. **Commit** using conventional commits (`feat:`, `fix:`, `chore:`, etc.)
5. **Open a PR** with a clear description of what and why

## What to Contribute

- **Bug fixes** — found something broken? Fix it.
- **New templates** — have a project type we don't cover? Add a template.
- **Agent adapters** — want Apollo to work with Cursor, Codex, Windsurf? Build an adapter.
- **Config schema improvements** — missing a preference that matters? Propose it.
- **Documentation** — typos, unclear sections, better examples.

## Guidelines

- Keep changes small and focused — one concern per PR
- Follow the existing code style (concise, minimal comments)
- Update the README if you change user-facing behavior
- Add a CHANGELOG entry for new features and fixes

## Reporting Issues

Open an issue with:
- What you expected to happen
- What actually happened
- Steps to reproduce
- Your environment (OS, agent, version)
