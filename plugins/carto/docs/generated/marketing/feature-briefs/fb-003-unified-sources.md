# Feature Brief: External Source Integration

## One-Liner

Enrich your codebase understanding with context from GitHub, Jira, Linear, Notion, Slack, and more.

## What It Is

Carto connects to the tools your team already uses and weaves that context into your codebase understanding. Pull requests explain how code evolved. Tickets explain why it was built. Documentation explains what's planned. Slack conversations capture decisions that never made it into docs.

Now your AI assistant knows all of it.

## Who It's For

**Primary:** Engineering teams whose context is scattered across multiple tools. The "why" behind the code lives in Jira tickets, the "how" lives in GitHub PRs, and the "what's next" lives in Notion docs.

**Secondary:** Engineering leaders who want AI assistants to understand the full development context, not just the code on disk.

## The Problem

Code alone doesn't tell the full story. A function exists because of a product requirement in Jira. An architectural decision was debated in a Slack thread. The planned refactor is outlined in a Notion document. A critical bug fix is documented in a GitHub PR.

When AI assistants only see code, they miss all of this. They can't explain why something was built a certain way, can't anticipate planned changes, and can't connect code to the business goals driving it.

## Key Benefits

- **Complete picture.** Code plus context gives AI assistants the understanding to make better suggestions, write more relevant code, and avoid contradicting decisions your team already made.
- **Automatic enrichment.** Carto pulls context from your tools without manual linking. It identifies connections between code and external sources automatically.
- **Eight source types out of the box.** GitHub (PRs, issues), Jira, Linear, Notion, Slack, and more. Each source type has a purpose-built connector that extracts the right information.
- **Always relevant.** Carto filters and prioritizes external context so AI assistants get signal, not noise. Recent, high-impact context surfaces first.

## How It Works (Simplified)

1. **Connect** -- Link your tools to Carto with API keys or OAuth.
2. **Enrich** -- Carto automatically pulls relevant context and associates it with the right parts of your codebase.
3. **Retrieve** -- When an AI assistant queries Carto, it gets code understanding enriched with the context that explains the "why" behind the code.

## Competitive Context

Other code intelligence tools index code in isolation. Carto is the only product that bridges the gap between code and the broader development ecosystem -- tickets, docs, PRs, and conversations -- delivering unified context to AI assistants.

**One-Liner:** "Code plus context. The full story, delivered to every AI assistant."
