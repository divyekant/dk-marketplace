# AI Skill Files: Instant Expertise for Every AI Assistant

## The Problem

AI coding assistants are only as good as the context they have. Without it, they produce generic code -- correct syntax, wrong patterns. They ignore your naming conventions, bypass your module boundaries, and reinvent solutions your team already standardized.

The fix is context files like CLAUDE.md and .cursorrules. But writing them by hand is tedious. Keeping them updated is worse. Most teams either have incomplete, outdated files or none at all. Every AI session starts with the same blind spots.

## The Solution

Carto automatically generates skill files from its deep analysis of your codebase. These files give Claude Code and Cursor instant, structured understanding of your project -- architecture, patterns, conventions, and domain logic -- in the exact format each tool expects.

No one writes them. No one maintains them. They stay current every time you re-index.

## Key Benefits

- **Automatic generation.** Carto builds skill files from its semantic analysis. No manual authoring, no template filling, no guesswork.
- **Native formats.** CLAUDE.md for Claude Code, .cursorrules for Cursor. Each file is structured exactly as the tool expects for maximum effectiveness.
- **Always up to date.** Re-index and skill files reflect your latest architecture, patterns, and conventions. No documentation drift.
- **Complete coverage.** Skill files include project architecture, module relationships, coding patterns, naming conventions, and domain-specific guidance. AI assistants get the full picture, not fragments.
- **Team-wide consistency.** Every developer's AI assistant reads the same context. Everyone gets the same quality of AI assistance, whether they joined yesterday or three years ago.

## How It Works

1. **Index** your codebase with Carto (takes under 90 seconds).
2. **Carto generates** CLAUDE.md and .cursorrules from its analysis.
3. **Drop the files** in your project root. AI assistants read them automatically.
